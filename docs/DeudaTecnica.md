## 2. Mejoras de diseño futuras
- **Automatización de Pruebas**: Configurar un framework de pruebas automatizadas (como Jest y Supertest) para los endpoints de la API en lugar de depender únicamente de verificaciones manuales en Thunder Client. Esto garantizará que cambios futuros en el modelo de base de datos no rompan la funcionalidad existente (regresión).
- **Notificaciones del Sistema y Sonido**: Agregar notificaciones nativas del navegador (Push Notifications) y alertas sonoras de baja frecuencia al culminar un temporizador de Modo Arranque o de Tarea, mejorando la inmersión del estudiante en sus periodos de estudio.
- **Sincronización Mascota-Tareas (Gamerización Completa)**: Conectar el sistema de tareas con el de mascotas virtuales de manera que al completar subtareas o sesiones de Modo Arranque, la mascota activa del usuario gane puntos de experiencia (EXP) o recupere puntos de salud (HP), fortaleciendo la motivación intrínseca.
