# Casos de Prueba – US-09

| ID | Qué se debe hacer (acción / entrada) | Salida esperada |
|-------|---------------------------------------------|--------------------------------|
| CP-01 | **[CRUD - Registro/Creación]** Enviar una petición POST a `/api/usuarios/1/mascota` con el cuerpo JSON: `{"nombre_mascota": "Firulais", "tipo_mascota": "Perro"}` (usuario 1 no tiene mascota). | Respuesta HTTP 201 Created con el objeto JSON de la mascota creada conteniendo: `id`, `usuario_id: 1`, `nombre_mascota: "Firulais"`, `tipo_mascota: "Perro"`, `nivel: 1`, `salud: 100`. |
| CP-02 | **[CRUD - Lectura/Consulta]** Enviar una petición GET a `/api/usuarios/1/mascota` después de haber creado la mascota en CP-01. | Respuesta HTTP 200 OK con el objeto JSON: `{"hasPet": true, "mascota": { ... }}` con los datos de la mascota del usuario 1. |
| CP-03 | **[CRUD - Eliminación]** Enviar una petición DELETE a `/api/usuarios/1/mascota` para liberar o borrar la mascota asociada al usuario 1. | Respuesta HTTP 200 OK con el cuerpo JSON: `{"mensaje": "Mascota eliminada exitosamente"}`. |
| CP-04 | **[Búsqueda con filtro (de tipo)]** En el frontend, seleccionar el filtro de tipo de mascota "Gato" e ingresar el nombre "Michi", luego presionar el botón de creación. | El sistema valida que "Gato" es un filtro de tipo válido y permite crear la mascota, retornando HTTP 201. |
| CP-05 | **[Validación de tipo inválido]** Enviar una petición POST a `/api/usuarios/4/mascota` con un tipo no soportado por el filtro: `{"nombre_mascota": "Rex", "tipo_mascota": "Dragón"}`. | Respuesta HTTP 400 Bad Request con el cuerpo JSON: `{"error": "El tipo de mascota debe ser uno de los siguientes: Perro, Gato, Hámster"}`. |
| CP-06 | **[Transacción y Relación de Entidades]** Intentar registrar una mascota para un usuario inexistente (id=999) enviando una petición POST a `/api/usuarios/999/mascota` con `{"nombre_mascota": "Copito", "tipo_mascota": "Gato"}`. | El servicio verifica la tabla `usuarios` y, al no encontrar la clave primaria correspondiente, rechaza la operación retornando HTTP 404 Not Found con el cuerpo JSON: `{"error": "Usuario no encontrado"}` (garantiza la integridad referencial de la relación Usuario-Mascota). |
| CP-07 | **[Transacción de unicidad - Límite de mascota por usuario]** Enviar una petición POST a `/api/usuarios/1/mascota` cuando el usuario 1 ya posee una mascota asignada en la base de datos. | El sistema verifica la existencia de una fila asociada en la tabla `mascotas` y cancela la transacción de inserción, respondiendo HTTP 400 Bad Request con el cuerpo JSON: `{"error": "El usuario ya tiene una mascota asignada"}`. |

Nota: incluir al menos un caso por cada operación relevante de la HU (búsqueda con filtro, CRUD y transacción que une entidades).
